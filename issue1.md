# Issue 1: 文の分割処理における誤検出の修正

## 問題の概要

README.mdで「1文中に読点が4個使用されています（閾値: 4個）」という警告が出ているが、これは誤検出です。文の分割処理（`sentenceParser.ts`）が、Markdown内の複数の段落や要素を1つの文として誤って結合しているため、読点のカウントが不正確になっています。

## 具体的な問題箇所

README.md の74行目付近：
```
対象となる接続詞: しかし、また、そして、それで、だから、ところが、すると、それから、さらに
```

この行には読点が8個ありますが、警告メッセージでは「4個」と表示されています。これは、文の分割処理が前後の段落と誤って結合しているためと推測されます。

## 修正してほしいこと

1. **`server/src/grammar/sentenceParser.ts`の調査**
   - 現在の文分割ロジックを確認
   - Markdown特有の要素（見出し、太字、リスト、表など）の扱いを確認
   - 改行と句点の扱いを確認

2. **文分割ロジックの修正**
   - Markdown見出し（`####`など）の後は必ず文の区切りとする
   - 空行は文の区切りとする
   - 太字マーカー（`**`）で囲まれた単独行は独立した要素として扱う
   - 表の前後は文の区切りとする
   - コロン（`:`）で終わる行の後は文の区切りとする（リストの導入部など）

3. **テストの追加**
   - Markdown特有の構造での文分割が正しく動作することを確認するテストを追加
   - 特に、見出し・空行・表・リストが含まれる複雑なMarkdownでのテスト

4. **既存テストの確認**
   - 修正により既存のテストが壊れていないか確認
   - `server/src/grammar/sentenceParser.test.ts`を実行

## 期待される結果

- 「対象となる接続詞: しかし、また、...」の行が独立した文として認識される
- または、コロンで終わる導入部として文法チェックの対象外になる
- 誤った読点カウントによる警告が出なくなる
